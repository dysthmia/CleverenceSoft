# LogStandardizerApp

Консольная программа для стандартизации лог-файлов.
Программа читает входной файл с логами в двух разных форматах, приводит записи к единому формату и сохраняет результат в отдельный файл.
Корректные строки записываются в `output.txt`, а строки с ошибками или неподходящим форматом записываются в `problems.txt`.

## Как запустить
Откройте терминал и перейдите в папку проекта:
```bash
cd LogStandartApp
dotnet run
```

## Как использовать
После запуска программы появится меню:

```text
=== Стандартизация лог-файлов ===
Текущий входной файл: input.txt
Текущий выходной файл: output.txt
Файл проблемных строк: problems.txt

1. Показать файлы в текущей папке
2. Выбрать входной файл
3. Указать выходной файл
4. Обработать файл
5. Создать пример input.txt
0. Выход

Выберите пункт:
```

## Описание пунктов меню

### 1. Показать файлы в текущей папке
Показывает все файлы, которые находятся в текущей папке проекта.
Обычно это папка, из которой была запущена программа. 
Этот пункт нужен, чтобы быстро посмотреть, какие файлы доступны для обработки.

### 2. Выбрать входной файл
Позволяет указать файл, из которого программа будет читать логи.
По умолчанию используется файл:
```text
input.txt
```
Если нужно обработать другой файл, выберите пункт `2` и введите путь к нему.

### 3. Указать выходной файл
Позволяет изменить файл, в который будут записываться корректно обработанные логи.
По умолчанию используется файл:
```text
output.txt
```

### 4. Обработать файл
Запускает обработку выбранного входного файла.
Программа читает файл построчно:
- если строка подходит под один из двух форматов логов, она преобразуется и записывается в выходной файл;
- если строка не подходит под нужный формат, она записывается в `problems.txt`.
После успешной обработки программа выведет сообщение:
```text
Файл успешно обработан.
Результат: output.txt
Невалидные строки: problems.txt
```

### 5. Создать пример input.txt
Создаёт тестовый файл `input.txt` с несколькими примерами логов.
Этот пункт нужен, чтобы можно было быстро проверить работу программы без ручного создания файла.
После выбора пункта `5` программа создаст файл `input.txt`, который можно сразу обработать через пункт `4`.

### 0. Выход
Завершает работу программы.
